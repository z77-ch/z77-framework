## mental model

The system's language policy has a single source of truth: the dedicated
`config/i18n` config (ADR-013), carrying `defaultLanguage` (the one system-wide
fallback) and `languages` (the system-wide routing whitelist). The installer
generates `config/i18n.inc.php` from the `core-i18n` installer-config section
(defaults in `i18n.default.inc.php`), like `auth` / `moduleManager`. At runtime the
`I18n` service (DI key `I18n`, registered in `Bootstrap::pullUp()`) reads it;
consumers reach it via `DI::getI18n()`.

- `defaultLanguage` replaced the former scattered constants. "Set everything to `en`"
  = change this one key.
- **Resolution + persistence (revised 2026-06-08, ADR-015):** `Request::extractLanguage()`
  accepts a leading two-letter URL segment only if it is in `languages` (unknown â†’
  `InvalidRouteException`). **The rendered language of a resolved URL is determined by the
  URL alone** â€” a prefix sets it, its absence means `defaultLanguage`. The session NEVER
  overrides the rendered language of a resolved page (doing so made the prefix-less canonical
  `/home` render in whatever language the session held â†’ unstable canonical / page-cache key).
  A valid prefix is the explicit choice and is persisted to the session. The remembered
  preference is used **only to redirect the bare site root**: `/` + a non-default remembered
  language â†’ 302 to `/{lang}`. `Request::applyLanguageSession(): ?string` returns that redirect
  target (else null); it runs in `Dispatcher::execute()` after `AccessGuard` started the session
  and before the page-cache decision, and the dispatcher issues the `RedirectResponse`.
- **Per-module switch (opt-in):** the global `languages` is the routing whitelist;
  a module's `languages` key (`App/Config/{module}Config.inc.php`, read via
  `ModuleManager::getModuleLanguages()`) is the switch offering for that environment.
  Omitted â†’ no switch (admin backend stays single-language). The frontend declares
  `['de','fr']`. `AbstractBaseController::html()` builds `$languageSwitch` (filtered
  to the global whitelist; only when > 1 language) and injects it into every template;
  `partials/header.tpl.php` renders it in the topbar (`fe-topbar__nav--lang` /
  `fe-topbar__link--lang` â€” a compact uppercase group with a left divider) and in the
  mobile overlay (`fe-nav-overlay__list--lang`, a divided horizontal row). Styles live
  in the frontend SCSS (`_topbar.scss` / `_nav-overlay.scss` â†’ `base.css`).
- **Switch link shape:** each entry is the current page's **canonical** form re-prefixed â€”
  `AbstractBaseController::currentCanonicalPath()` (the canonical NavigationAlias path of
  `$current` + content slugs, ADR-015) localized for the target code (always prefixed, incl.
  the default, so a click explicitly sets + persists). NOT the as-requested path â€” a
  non-canonical alias still switches to the canonical localized URL.
- **Content language fallback:** `ContentService::find()` falls back to the
  `defaultLanguage` document when the requested-language document does not exist; an
  existing-but-inactive document does NOT fall back.
- The generated `skeleton/config/i18n.inc.php` and `skeleton/data/content/*.fr.json`
  are RUNTIME and ephemeral â€” regenerated/seeded by the installer from the SOURCE
  defaults; do not hand-edit.

## rules

- When needing the system default language in PHP â†’ MUST read `DI::getI18n()->getDefaultLanguage()`; MUST NOT hardcode `'de'` or introduce a local `DEFAULT_LANGUAGE` constant.
- When accepting a language code from the URL or a request â†’ MUST validate it with `I18n::isValidLanguage()`; MUST NOT accept an arbitrary two-letter segment.
- When changing the system default language ("set all to X") â†’ MUST change `defaultLanguage` in the `config/i18n` source only; MUST NOT edit it in a module config or in code.
- When adding a globally servable language â†’ MUST add it to `config/i18n.languages` via the installer-config source (`core-i18n`); MUST NOT hand-edit the generated `skeleton/config/i18n.inc.php`.
- When an environment should offer a language switch â†’ MUST declare `languages` (a subset of the global whitelist) in its own `App/Config/{module}Config.inc.php`; omitting the key MUST mean no switch (opt-in); MUST NOT rely on inheriting the global set.
- When building a language-switch link in a template â†’ MUST use the injected `$languageSwitch` items (`code`/`url`/`active`); MUST NOT construct language URLs ad-hoc in the template.
- When loading frontend content in a language â†’ MUST go through `ContentService::find()`/`render()` so the default-language fallback applies; MUST NOT read the repository per-language directly on the frontend.

## known issues

- Don't assume the page cache is shared across languages: the page cache keys per language (`PageIdentity` includes `language`). A visitor gets a session only to remember a switch preference for the root redirect (below) â€” the rendered language itself is URL-derived.

- **I18N-LANG-STABLE-001** â€” resolved 2026-06-08 (ADR-015, Variante B). The prefix-less canonical URL was not language-stable: `applyLanguageSession()` adopted the session language for any prefix-less request, so `/home` rendered de or fr depending on the session (broken canonical + wrong-language page-cache entry under the de key). Fix: a prefix-less resolved URL ALWAYS renders `defaultLanguage`; the session only redirects the bare root `/` â†’ `/{lang}`. `applyLanguageSession()` now returns the optional redirect target; `Dispatcher` issues it. Supersedes ADR-013 Â§"Language resolution & persistence" (the "prefix-less adopts remembered language" clause). Verified: fr-session + `/home` â†’ `<html lang="de">`; fr-session + `/` â†’ 302 `/fr`.

- **Per-module subset validation** â€” resolved 2026-06-07. `getModuleLanguages()` now fails fast: a module that declares a non-empty `languages` set MUST include the system `defaultLanguage` (the guaranteed fallback). Omitting it throws a `RuntimeException` at the consumption boundary instead of silently relying on the implicit fallback. An empty/absent declaration still means "no switch" (opt-out), no assertion.
